=====================================

第一章 系统概述

QQ-Claude-Proxy 是一个通过 QQ 远程控制本地 Claude Code CLI 的代理系统。
用户通过手机 QQ 发送消息，消息经过 QQ 开放平台传送到本地服务器，
本地服务器调用 claude CLI 执行命令，然后将结果返回给 QQ。

第二章 核心架构

项目采用三层架构，通过 WebSocket Gateway 实现组件解耦：

QQ Bot → QQ Gateway ──────┐
                          │
                    Internal Gateway (WS, port 18789)
                          │
                          ├──→ Agent (Claude Code CLI)
                          └──→ 可扩展其他 Channel

第三章 功能特点

1. 技能管理 - 通过 QQ 安装、卸载、搜索技能
2. 会话持久化 - 服务重启后自动恢复对话状态
3. 视觉理解 - 图像分析和理解能力
4. 多 Agent 协调 - 支持团队模式和 CLI 模式切换

第四章 支持的文件格式

文本格式：
- .txt - 纯文本文件
- .md - Markdown 文档

第五章 测试说明

本文件用于测试大文件的分段发送功能。
当文件内容超过 3000 字符时，系统会自动分段发送，
每段最多 2000 字符。

第六章 技术实现

分段发送通过以下方式实现：
1. 读取完整文件内容
2. 检查文件大小
3. 如果超过限制，使用正则表达式分段
4. 逐段发送，添加段号和总段数
5. 段与段之间添加延迟，避免频率限制

第三段内容：
=====================================

第七章 使用示例

1. 小文件发送（< 3000 字符）
   直接发送，不分段

2. 大文件发送（> 3000 字符）
   自动分段，每段 2000 字符

第八章 注意事项

1. 分段发送时，每段都会添加文件名和段号
2. 段与段之间有 500ms 延迟
3. 如果某段发送失败，会停止发送后续段

- 文件队列管理
- 分段发送机制
- 错误处理和恢复
