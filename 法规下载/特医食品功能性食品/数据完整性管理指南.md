# 数据完整性管理指南

## 指南基本信息

| 项目 | 内容 |
|------|------|
| 制定机构 | 国家药品监督管理局 |
| 发布年份 | 2024年 |
| 适用范围 | 药品、特医食品、保健食品生产企业 |
| 指南性质 | 技术指导原则 |

## 1. 总则

### 1.1 目的

为规范药品、特殊医学用途配方食品、保健食品生产企业数据管理，确保数据真实、准确、完整和可靠，保障产品质量和安全，制定本指南。

### 1.2 适用范围

本指南适用于：
1. 特殊医学用途配方食品生产企业
2. 保健食品生产企业
3. 药品生产企业
4. 上述企业的质量管理活动

### 1.3 法律依据

- 《中华人民共和国药品管理法》
- 《中华人民共和国食品安全法》
- 《药品生产质量管理规范》
- 《药品记录与数据管理要求》

## 2. 基本原则

### 2.1 ALCOA+原则

数据完整性应遵循ALCOA+原则：

| 原则 | 英文 | 含义 | 具体要求 |
|------|------|------|----------|
| **可归因** | Attributable | 数据来源可追溯 | 记录谁、何时、为何产生 |
| **清晰可读** | Legible | 记录清晰易读 | 永久保存，不得涂改 |
| **同步记录** | Contemporaneous | 边做边记 | 实时记录，不得补记 |
| **原始记录** | Original | 保留原始数据 | 保留原始载体和格式 |
| **准确** | Accurate | 数据准确无误 | 真实可靠，计算正确 |
| **完整** | Complete | 数据完整无缺失 | 含全部信息和附件 |
| **一致** | Consistent | 数据逻辑一致 | 前后一致，逻辑合理 |
| **持久** | Enduring | 数据长期保存 | 保存介质耐用 |
| **可获取** | Available | 数据便于检索 | 快速调取，随时可用 |

### 2.2 数据分级

根据数据对产品质量的影响程度进行分级：

| 级别 | 定义 | 管理要求 |
|------|------|----------|
| **关键数据** | 直接影响产品质量安全 | 最严格管理 |
| **重要数据** | 间接影响产品质量安全 | 严格管理 |
| **一般数据** | 支持性数据 | 常规管理 |

## 3. 数据类型

### 3.1 按载体分类

| 类型 | 说明 | 管理要点 |
|------|------|----------|
| **纸质记录** | 纸质载体记录 | 笔迹、修改、保存 |
| **电子记录** | 计算机系统记录 | 权限、审计追踪、备份 |
| **混合记录** | 纸质+电子 | 一致性、同步 |

### 3.2 按内容分类

| 类型 | 示例 | 保存期限 |
|------|------|----------|
| **批记录** | 批生产记录、批检验记录 | 产品有效期后1年 |
| **设备记录** | 运行、维护、校准记录 | 设备报废后3年 |
| **质量记录** | 偏差、变更、CAPA | 长期 |
| **人员记录** | 培训、健康档案 | 在职期间+3年 |
| **验证记录** | 验证方案、报告 | 验证对象生命周期 |

## 4. 纸质记录管理

### 4.1 记录要求

#### 4.1.1 基本要求

| 要求 | 说明 | 示例 |
|------|------|------|
| 使用蓝色或黑色墨水 | 不得使用铅笔或可擦笔 | 蓝黑钢笔、黑色签字笔 |
| 字迹清晰工整 | 易于辨认 | 正楷书写 |
| 不得涂改 | 错误修改规范 | 单线划去+签名+日期 |
| 及时记录 | 边做边记 | 操作完成立即记录 |
| 签名全名 | 不得简签 | 张三不得签张 |
| 填写日期 | 完整日期 | 2024-02-28 |

#### 4.1.2 修改规范

```
错误记录修改流程：
1. 单线划去错误内容（保持可读）
2. 在上方写上正确内容
3. 签名并注明修改日期
4. 简要说明修改原因

示例：
温度：25°C  30°C
          张三 2024-02-28 原记录笔误
```

#### 4.1.3 空白处理

| 情况 | 处理方式 |
|------|----------|
| 数据项不适用 | 填写"N/A"或"不适用" |
| 无数据 | 划单横线"一"或填"0" |
| 留有空白行 | 划单横线"一"并签名 |

### 4.2 记录控制

#### 4.2.1 记录发放

| 项目 | 要求 |
|------|------|
| 受控发放 | 按编号发放 |
| 领用登记 | 领用人签字 |
| 序号管理 | 唯一编号 |
| 作废回收 | 作废记录回收 |

#### 4.2.2 记录填写

| 项目 | 要求 |
|------|------|
| 及时 | 操作完成后立即填写 |
| 准确 | 数据真实准确 |
| 完整 | 所有项目填写齐全 |
| 清晰 | 字迹清晰易读 |
| 签名 | 操作人、复核人签名 |

#### 4.2.3 记录审核

| 审核类型 | 审核内容 | 审核要求 |
|----------|----------|----------|
| 复核 | 数据准确性 | 复核人签名 |
| 审核 | 合规性检查 | 审核人签名 |
| 批准 | 最终批准 | 批准人签名 |

### 4.3 记录保存

#### 4.3.1 保存条件

| 要求 | 说明 |
|------|------|
| 环境 | 防潮、防火、防虫、防鼠 |
| 装订 | 按顺序装订成册 |
| 归档 | 分类归档，标识清晰 |
| 期限 | 按法规要求保存 |

#### 4.3.2 保存期限

| 记录类型 | 保存期限 | 备注 |
|----------|----------|------|
| 批生产记录 | 产品有效期后1年 | 最短不少于5年 |
| 批检验记录 | 产品有效期后1年 | 最短不少于5年 |
| 验证记录 | 设备使用年限或长期 | 关键验证永久保存 |
| 校准记录 | 5年 | |
| 维护记录 | 3年 | |
| 人员培训记录 | 在职期间+3年 | |
| 偏差处理记录 | 3年 | |
| 变更控制记录 | 永久 | |

## 5. 电子记录管理

### 5.1 计算机化系统要求

#### 5.1.1 系统验证

| 验证阶段 | 验证内容 | 文件 |
|----------|----------|------|
| **URS** | 用户需求说明 | 用户需求说明书 |
| **DQ** | 设计确认 | 设计确认报告 |
| **IQ** | 安装确认 | 安装确认报告 |
| **OQ** | 运行确认 | 运行确认报告 |
| **PQ** | 性能确认 | 性能确认报告 |

#### 5.1.2 功能要求

| 功能 | 要求 |
|------|------|
| 用户管理 | 独立账户、权限分级 |
| 密码控制 | 复杂度要求、定期更换 |
| 电子签名 | 唯一、不可伪造、可追溯 |
| 审计追踪 | 记录所有操作 |
| 数据备份 | 定期备份、异地存储 |
| 数据恢复 | 可靠的恢复机制 |

### 5.2 用户权限管理

#### 5.2.1 权限分级

| 级别 | 权限 | 职责 |
|------|------|------|
| 系统管理员 | 系统配置、用户管理 | IT人员 |
| 数据管理员 | 数据审核、批准 | 质量人员 |
| 数据录入员 | 数据录入、修改 | 操作人员 |
| 数据浏览员 | 仅查看 | 一般人员 |

#### 5.2.2 密码要求

| 要求 | 说明 |
|------|------|
| 长度 | 不少于8位 |
| 复杂度 | 大小写字母+数字+符号 |
| 有效期 | 90天更换一次 |
| 历史 | 不得重复最近5次 |
| 锁定 | 连续5次失败锁定 |

### 5.3 电子签名

#### 5.3.1 签名要求

| 要求 | 说明 |
|------|------|
| 唯一性 | 一人一签，不得共用 |
| 不可伪造 | 密码或生物识别 |
| 可追溯 | 签名+时间戳 |
| 绑定 | 签名与操作绑定 |

#### 5.3.2 签名效力

电子签名与手写签名具有同等法律效力，需满足：
1. 签名者身份真实
2. 签名行为自愿
3. 签名后不可修改
4. 签名可验证

### 5.4 审计追踪

#### 5.4.1 追踪内容

| 操作类型 | 记录内容 |
|----------|----------|
| 登录/登出 | 用户、时间、IP |
| 数据创建 | 创建人、时间、数据 |
| 数据修改 | 修改人、时间、原值、新值、原因 |
| 数据删除 | 删除人、时间、被删数据、原因 |
| 数据导出 | 导出人、时间、范围 |
| 权限变更 | 变更人、时间、变更内容 |

#### 5.4.2 追踪要求

| 要求 | 说明 |
|------|------|
| 自动记录 | 系统自动记录 |
| 不可修改 | 审计追踪不可修改 |
| 长期保存 | 与数据同步保存 |
| 定期审查 | 定期审查异常操作 |

### 5.5 数据备份与恢复

#### 5.5.1 备份策略

| 备份类型 | 频率 | 保留时间 |
|----------|------|----------|
| 完全备份 | 每周 | 1个月 |
| 增量备份 | 每日 | 1周 |
| 差异备份 | 每日 | 1周 |
| 异地备份 | 每月 | 1年 |

#### 5.5.2 备份验证

| 验证内容 | 频率 |
|----------|------|
| 备份完整性检查 | 每月 |
| 恢复测试 | 每季度 |
| 异地备份检查 | 每年 |

#### 5.5.3 恢复要求

| 要求 | 目标 |
|------|------|
| RTO（恢复时间目标） | ≤4小时 |
| RPO（恢复点目标） | ≤1天 |
| 数据一致性 | 100% |

## 6. 数据风险管控

### 6.1 风险识别

| 风险类型 | 风险描述 | 风险等级 |
|----------|----------|----------|
| 数据造假 | 故意伪造数据 | 高 |
| 数据篡改 | 修改历史数据 | 高 |
| 数据丢失 | 备份失效、系统故障 | 高 |
| 数据错误 | 操作失误、系统错误 | 中 |
| 数据泄露 | 未授权访问 | 中 |
| 数据不一致 | 多系统数据不同步 | 低 |

### 6.2 风险评估

| 评估维度 | 评估方法 |
|----------|----------|
| 可能性 | 历史、经验、统计 |
| 严重性 | 影响范围、后果 |
| 可检测性 | 检测难度 |
| 风险值 | 可能性×严重性 |

### 6.3 风险控制

| 控制措施 | 适用风险 |
|----------|----------|
| 权限管理 | 数据篡改、泄露 |
| 审计追踪 | 数据造假、篡改 |
| 数据备份 | 数据丢失 |
| 定期审查 | 数据错误 |
| 系统验证 | 系统错误 |
| 人员培训 | 操作失误 |

## 7. 数据质量监控

### 7.1 监控指标

| 指标 | 计算方法 | 目标值 |
|------|----------|--------|
| 数据准确率 | 准确数据量/总数据量×100% | ≥99.9% |
| 数据完整率 | 完整记录数/总记录数×100% | 100% |
| 数据及时率 | 及时记录数/总记录数×100% | ≥95% |
| 审计追踪覆盖率 | 有追踪操作数/总操作数×100% | 100% |
| 备份成功率 | 成功备份次数/总备份次数×100% | 100% |

### 7.2 数据审查

#### 7.2.1 定期审查

| 审查类型 | 频率 | 内容 |
|----------|------|------|
| 日常审查 | 每日 | 当日数据完整性 |
| 周审查 | 每周 | 一周数据质量 |
| 月审查 | 每月 | 月度数据趋势 |
| 年审查 | 每年 | 全年数据质量评估 |

#### 7.2.2 审查方法

| 方法 | 适用场景 |
|------|----------|
| 抽样检查 | 大数据量 |
| 全面审查 | 关键数据 |
| 对比检查 | 跨系统数据 |
| 趋势分析 | 数据质量趋势 |

## 8. 不符合处理

### 8.1 问题分类

| 分类 | 定义 | 处理时限 |
|------|------|----------|
| 严重问题 | 影响产品质量安全 | 立即 |
| 重要问题 | 可能影响产品质量安全 | 3天内 |
| 一般问题 | 不影响产品质量安全 | 1周内 |

### 8.2 处理流程

```
问题发现
    ↓
问题报告
    ↓
原因分析
    ↓
风险评估
    ↓
纠正措施
    ↓
预防措施
    ↓
效果验证
    ↓
关闭
```

### 8.3 预防措施

| 措施 | 说明 |
|------|------|
| 系统优化 | 优化系统功能 |
| 流程完善 | 完善操作流程 |
| 培训加强 | 加强人员培训 |
| 制度修订 | 修订管理制度 |
