# 文件变更控制程序

## 程序基本信息

| 项目 | 内容 |
|------|------|
| 制定机构 | 企业质量管理部门 |
| 适用范围 | 质量管理体系文件变更控制 |
| 程序性质 | 企业内部管理程序 |

## 1. 总则

### 1.1 目的

规范质量管理体系文件变更控制，确保文件变更经过充分评估和批准，防止未经授权的变更，保证体系文件的有效性。

### 1.2 适用范围

本程序适用于：
1. 质量手册变更
2. 程序文件变更
3. 作业指导书变更
4. 记录表单变更
5. 技术标准变更

### 1.3 基本原则

| 原则 | 说明 |
|------|------|
| **必要性** | 变更必须有必要 |
| **充分评估** | 充分评估变更影响 |
| **严格审批** | 严格审批流程 |
| **及时实施** | 及时实施变更 |
| **有效培训** | 确保人员掌握 |

## 2. 文件分类

### 2.1 按层级分类

| 层级 | 文件类型 | 变更审批权限 |
|------|----------|--------------|
| 一级 | 质量手册 | 质量负责人 |
| 二级 | 程序文件 | 部门负责人 |
| 三级 | 作业指导书 | 班组长/主管 |
| 四级 | 记录表单 | 部门负责人 |

### 2.2 按性质分类

| 性质 | 说明 | 示例 |
|------|------|------|
| 管理性文件 | 管理制度、程序 | 质量手册、程序文件 |
| 技术性文件 | 技术标准、规程 | 工艺规程、检验规程 |
| 操作性文件 | 操作指导 | SOP、作业指导书 |
| 记录性文件 | 记录表单 | 各种记录表格 |

### 3.1 变更申请条件

| 情况 | 说明 |
|------|------|
| 法规更新 | 法规标准更新 |
| 工艺改进 | 工艺技术改进 |
| 设备更新 | 设备更新改造 |
| 质量问题 | 质量问题纠正 |
| 客户要求 | 客户特殊要求 |
| 内部审核 | 内审发现不符合 |

### 3.2 变更申请表

| 项目 | 内容 |
|------|------|
| 申请编号 | 唯一编号 |
| 申请日期 | 申请时间 |
| 申请人 | 申请人姓名 |
| 部门 | 所属部门 |
| 文件编号 | 变更文件编号 |
| 文件名称 | 变更文件名称 |
| 版本号 | 当前版本号 |
| 变更类型 | 修订/新增/废止 |
| 变更原因 | 详细说明变更原因 |
| 变更内容 | 详细说明变更内容 |
| 影响评估 | 初步评估变更影响 |
| 申请人签字 | 签字 |
| 日期 | 签字日期 |

### 3.3 变更类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **新增** | 新增文件 | 新增程序文件 |
| **修订** | 修改现有文件 | 修订SOP |
| **替换** | 整体替换文件 | 换版 |
| **废止** | 废止文件 | 废止过时文件 |

## 4. 变更评估

### 4.1 评估内容

| 评估项目 | 说明 |
|----------|------|
| **必要性** | 变更是否必要 |
| **可行性** | 变更是否可行 |
| **合规性** | 是否符合法规 |
| **影响范围** | 影响哪些文件/部门 |
| **资源需求** | 需要哪些资源 |
| **风险** | 有哪些风险 |

### 4.2 影响评估

| 影响类别 | 评估内容 |
|----------|----------|
| 质量影响 | 对产品质量的影响 |
| 合规影响 | 对法规符合性的影响 |
| 运营影响 | 对生产运营的影响 |
| 成本影响 | 成本变化 |
| 培训需求 | 需要培训的人员 |

### 4.3 评估流程

```
变更申请
    ↓
初步评估（申请人）
    ↓
部门审核（部门负责人）
    ↓
相关部门会签
    ↓
质量部审核
    ↓
风险评估（如需要）
    ↓
审批
```

## 5. 变更审批

### 5.1 审批权限

| 文件层级 | 审批人 | 备注 |
|----------|--------|------|
| 质量手册 | 质量负责人/总经理 | 需要总经理批准 |
| 程序文件 | 部门负责人+质量负责人 | 双重审批 |
| 作业指导书 | 部门负责人 | 单独审批 |
| 记录表单 | 部门负责人 | 单独审批 |

### 5.2 审批流程

| 审批环节 | 审批人 | 审批重点 |
|----------|--------|----------|
| 部门审核 | 部门负责人 | 业务合理性 |
| 会签审核 | 相关部门 | 跨部门影响 |
| 质量审核 | 质量部门 | 体系符合性 |
| 风险审核 | 风险管理（如需要） | 风险可控 |
| 最终审批 | 授权批准人 | 最终决策 |

### 5.3 审批时限

| 审批类型 | 时限 |
|----------|------|
| 常规审批 | 5个工作日 |
| 紧急审批 | 2个工作日 |
| 重大变更 | 10个工作日 |

## 6. 变更实施

### 6.1 实施准备

| 准备工作 | 说明 |
|----------|------|
| 人员准备 | 确定执行人员 |
| 培训准备 | 准备培训材料 |
| 资源准备 | 准备所需资源 |
| 时机选择 | 选择合适时机 |

### 6.2 培训实施

| 培训类型 | 说明 |
|----------|------|
| 变更培训 | 针对变更内容培训 |
| 操作培训 | 针对新操作培训 |
| 考核 | 培训后考核 |

### 6.3 文件发放

| 步骤 | 说明 |
|------|------|
| 1. 文件打印 | 按需要份数打印 |
| 2. 文件盖章 | 加盖"受控"章 |
| 3. 文件发放 | 发放至使用部门 |
| 4. 旧版回收 | 回收旧版文件 |
| 5. 发放记录 | 记录发放情况 |

### 6.4 实施日期

| 情况 | 实施日期 |
|------|----------|
| 正常变更 | 批准后1个月内 |
| 紧急变更 | 批准后立即 |
| 重大变更 | 批准后3个月内 |

| 问题收集 | 是否存在问题 |

| 人员访谈 | 访谈相关人员 |

| 效果评估 | 评估变更效果 |

### 7.3 验证报告

| 内容 | 说明 |
|------|------|
| 验证目的 | 验证目的 |

## 8. 变更关闭

### 8.1 关闭条件

| 条件 | 说明 |
|------|------|
| 实施完成 | 变更已实施 |
| 培训完成 | 相关人员已培训 |
| 验证合格 | 验证合格 |
| 文件更新 | 相关文件已更新 |
| 记录完整 | 记录完整 |

### 8.2 关闭流程

```
变更实施
    ↓
培训完成
    ↓
验证合格
    ↓
更新记录
    ↓
变更关闭
```

### 8.3 关闭文件

| 文件 | 保存期限 |
|------|----------|
| 变更申请 | 文件有效期+3年 |
| 审批记录 | 文件有效期+3年 |
| 培训记录 | 3年 |
| 验证报告 | 3年 |
