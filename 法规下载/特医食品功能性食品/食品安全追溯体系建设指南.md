## 一、总则

### （一）编制目的

为指导食品生产经营企业建立完善的食品安全追溯体系，实现食品来源可查、去向可追、责任可究，保障食品安全，制定本指南。

### （二）适用范围

本指南适用于食品生产经营企业建立和实施食品安全追溯体系。

重点适用范围：

1. 特殊食品生产企业
2. 大型食品生产企业
3. 连锁经营企业总部
4. 网络食品交易第三方平台
5. 食品集中交易市场开办者

### （三）基本原则

1. **企业主体责任原则**：企业是追溯体系建设的责任主体
2. **全过程追溯原则**：覆盖从原料到消费的全过程
3. **信息真实完整原则**：确保追溯信息真实、准确、完整
4. **技术可行原则**：采用成熟可靠的技术方案
5. **经济合理原则**：兼顾追溯效果和经济成本

### （四）建设目标

到2026年底，实现以下目标：

1. 重点食品品种基本实现全程追溯
2. 特医食品、保健食品100%实现追溯
3. 大型食品生产企业100%建立追溯体系
4. 追溯信息互联互通基本实现

## 二、追溯体系架构

### （一）总体架构

食品安全追溯体系采用"一码贯通、三级追溯、多方协同"的架构：

```
                    ┌─────────────────┐
                    │   国家追溯平台   │
                    └────────┬─────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
┌───────▼───────┐   ┌────────▼────────┐   ┌──────▼──────┐
│  企业追溯系统  │   │  地方追溯平台   │   │ 第三方平台  │
└───────┬───────┘   └────────┬────────┘   └──────┬──────┘
        │                    │                    │
        └────────────────────┼────────────────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
        ┌─────▼────┐  ┌─────▼────┐  ┌─────▼────┐
        │  原料供应  │  │ 生产加工  │  │ 流通销售  │
        └──────────┘  └──────────┘  └──────────┘
```

### （二）追溯层级

追溯体系分为三个层级：

1. **L1级（基础追溯）**
   - 记录产品基本信息
   - 记录原料来源信息
   - 记录产品流向信息

2. **L2级（过程追溯）**
   - 记录生产过程信息
   - 记录检验检测信息
   - 记录储存运输信息

3. **L3级（全程追溯）**
   - 实现全链条信息贯通
   - 实现正向追溯和反向追溯
   - 实现风险预警和应急响应

### （二）信息采集要求

1. **及时性**：信息应当在操作完成后24小时内录入
2. **准确性**：信息应当真实准确，不得虚假
3. **完整性**：必填信息不得遗漏
4. **规范性**：信息格式应当符合标准规范

### （三）信息存储要求

1. **存储介质**：采用可靠的存储介质
2. **备份要求**：定期备份，确保数据安全
3. **保存期限**：产品保质期届满后不少于6个月
4. **访问权限**：建立访问权限管理

## 四、追溯编码规则

### （一）编码原则

追溯编码应遵循以下原则：

1. **唯一性**：每个产品/批次有唯一编码
2. **稳定性**：编码一经确定不随意变更
3. **可扩展性**：编码规则应可扩展
4. **兼容性**：兼顾现有编码体系

### （二）编码结构

追溯编码推荐采用以下结构：

```
企业标识(6位) + 产品类别(2位) + 生产日期(8位) + 批次号(4位) + 流水号(4位)
```

示例：`123456012024022800010001`

- 123456：企业标识
- 01：产品类别（特医食品）
- 20240228：生产日期
- 0001：批次号
- 0001：流水号

### （三）条码应用

追溯编码应转换为条码/二维码：

1. **一维码**：采用GS1-128码制
2. **二维码**：采用QR Code码制
3. **数据载体**：印刷在产品包装或标签上
4. **识读要求**：保证识读率≥99%

## 五、追溯系统建设

### （一）系统功能要求

追溯系统应具备以下核心功能：

#### 1. 信息录入功能

- 原料信息录入
- 生产信息录入
- 检验信息录入
- 流通信息录入
- 支持批量导入

#### 2. 信息查询功能

- 正向查询（原料→成品）
- 反向查询（成品→原料）
- 批次查询
- 综合查询

#### 3. 追溯展示功能

- 追溯链条可视化
- 追溯信息展示
- 产品信息展示

#### 4. 预警功能

- 过期产品预警
- 不合格产品预警
- 召回产品预警

#### 5. 统计分析功能

- 生产统计
- 流向统计
- 问题产品统计

### （二）系统技术要求

1. **性能要求**
   - 并发用户≥100
   - 响应时间≤3秒
   - 系统可用性≥99%

2. **安全要求**
   - 数据加密传输
   - 访问权限控制
   - 操作日志记录
   - 数据备份恢复

3. **接口要求**
   - 支持与国家平台对接
   - 支持与地方平台对接
   - 支持标准数据交换

### （三）系统部署方式

企业可根据实际情况选择部署方式：

| 部署方式 | 优点 | 缺点 | 适用对象 |
|----------|------|------|----------|
| 本地部署 | 数据安全可控 | 投资大维护难 | 大型企业 |
| 云端部署 | 投资少易维护 | 数据依赖第三方 | 中小企业 |
| 混合部署 | 兼顾两者 | 架构复杂 | 大中型企业 |

## 六、追溯实施步骤

### （一）准备阶段

1. **组织准备**
   - 成立追溯体系建设工作组
   - 明确职责分工
   - 制定实施方案

2. **资源准备**
   - 落实建设资金
   - 准备硬件设备
   - 培训操作人员

3. **技术准备**
   - 选择技术方案
   - 确定编码规则
   - 设计追溯流程

### （二）建设阶段

1. **系统开发/采购**
   - 自主开发或采购系统
   - 系统安装调试
   - 功能测试验证

2. **数据准备**
   - 整理历史数据
   - 建立数据标准
   - 数据迁移导入

3. **流程对接**
   - 修改现有流程
   - 增加追溯环节
   - 人员培训上岗

### （三）试运行阶段**

1. **试点运行**
   - 选择部分产品试点
   - 验证追溯功能
   - 发现问题改进

2. **全面推广**
   - 扩大试点范围
   - 逐步推广到全部产品
   - 持续优化完善

### （四）正式运行**

1. **系统正式上线**
   - 正式启用追溯系统
   - 停止旧系统运行
   - 数据备份归档

2. **运维保障**
   - 建立运维制度
   - 保障系统稳定运行
   - 定期系统升级

## 七、追溯管理要求

### （一）制度管理

企业应当建立追溯管理制度，包括：

1. 追溯信息采集制度
2. 追溯信息管理制度
3. 追溯系统操作规程
4. 追溯应急处理预案

### （二）人员管理

1. **人员配备**
   - 指定追溯管理负责人
   - 配备追溯操作人员
   - 明确岗位职责

2. **人员培训**
   - 系统操作培训
   - 业务知识培训
   - 定期复训

### （三）数据管理

1. **数据录入**
   - 及时录入追溯信息
   - 确保信息准确完整
   - 建立审核机制

2. **数据维护**
   - 定期备份数据
   - 清理无效数据
   - 优化数据结构

3. **数据安全**
   - 建立访问权限
   - 加密敏感数据
   - 防止数据泄露

### （四）应急管理

1. **应急响应**
   - 发现问题立即启动应急预案
   - 快速定位问题产品
   - 及时采取控制措施

2. **追溯演练**
   - 定期组织追溯演练
   - 验证追溯功能
   - 完善应急预案

## 八、监督检查

### （一）企业自查

企业应当定期对追溯体系进行自查：

1. 系统运行状况
2. 信息录入完整性
3. 追溯功能有效性
4. 人员培训情况

### （二）监管检查

市场监督管理部门应当：

1. 将追溯体系建设纳入日常检查
2. 检查追溯信息录入情况
3. 验证追溯功能有效性
4. 对不合规企业依法处理

### （三）第三方评估

鼓励企业委托第三方机构进行追溯体系评估：

1. 评估体系建设情况
2. 评估系统运行效果
3. 提出改进建议

## 九、附则

### （一）术语解释

1. **追溯**：通过记录标识追溯产品历史、应用情况或所处位置的能力。

2. **正向追溯**：从原料到成品的追溯。

3. **反向追溯**：从成品到原料的追溯。

4. **追溯节点**：追溯链条上的关键环节。
