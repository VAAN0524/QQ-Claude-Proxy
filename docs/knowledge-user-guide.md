# 知识库快速使用指南

**版本**: v3.1 (图书馆目录式体验)
**更新**: 2026-04-24

---

## 🎯 核心特性

### 📚 图书馆目录式展示
- **目录预览**: 先看摘要，再选详情
- **标签分组**: 按工作/学习/项目分类
- **智能去重**: 自动合并重复内容

### 🔍 三种视图模式

#### 1. 目录视图（搜索结果）
```
/kb search <关键词>
```
- 📋 按标签分组的目录列表
- 📄 每条显示标题+摘要
- 🏷️ 显示三层标签
- 🆔 显示 ID（用于查看详情）

#### 2. 紧凑视图（快速列表）
```
/kb list [标签]
```
- 📋 简洁的单行列表
- 🏷️ 显示二级标签
- 适合小屏幕快速浏览

#### 3. 详情视图（完整内容）
```
/kb get <ID>
```
- 📖 显示完整内容
- 📊 显示元数据（使用次数、重要性等）
- ⭐ 显示重要性评分

## 🚀 常用命令

### 搜索知识
```bash
/kb search WebSocket        # 搜索关键词
/kb search 调试 React       # 多关键词搜索
```

**输出示例**:
```
📚 知识库目录 (共 2 条)
══════════════════════════════════════════

## 📂 工作 (2 条)

  📄 WebSocket连接超时的解决方法 [80%匹配]
     └─ 检查网络连接、调整超时时间、添加心跳机制...
     └─ 🏷️ 工作 > 调试
     └─ 🆔 123e4567...
     └─ 👁️ text

💡 提示: 使用 /kb get <ID> 查看完整内容
```

### 查看详情
```bash
/kb get 123e4567            # 使用完整的 ID
/kb get 123e               # 使用前几位即可
```

### 列出知识
```bash
/kb list                   # 列出所有
/kb list 工作              # 按标签筛选
```

### 保存知识
```bash
/kb save "内容" 工作 调试 WebSocket
```

### 查看统计
```bash
/kb stats                  # 知识库统计
/kb tags                   # 所有标签
```

### 自然语言交互
```bash
/kb chat 帮我找找关于调试的知识
/kb ask 保存一下：React Hooks最佳实践
```

## 💡 使用技巧

### 1. 快速定位知识

**场景**: 你记得以前保存过某个知识点，但记不清具体内容

**方法**:
```bash
/kb search 关键词         # 先搜索
# 从目录中找到相关条目
/kb get <ID>              # 查看详情
```

### 2. 按分类浏览

**场景**: 想看看某个分类下有哪些知识

**方法**:
```bash
/kb list 工作             # 列出工作相关的知识
/kb list 学习             # 列出学习相关的知识
```

### 3. 了解知识库状态

**场景**: 想知道知识库中有多少内容

**方法**:
```bash
/kb stats
```

**输出示例**:
```
📊 知识库统计

总条目数: 25

按类型:
  - text: 25

按一级标签:
  - 工作: 15
  - 学习: 8
  - 项目: 2

最常用:
  1. WebSocket连接超时... (5 次)
  2. React Hooks性能... (3 次)
```

### 4. 避免重复保存

**场景**: 不确定某个知识是否已经保存过

**方法**:
```bash
/kb search 关键词         # 先搜索
# 如果找到了，就不再保存
# 如果没找到，再保存
/kb save "新知识" 标签1 标签2 标签3
```

## 🏷️ 三层标签体系

### 标签结构
```
一级（大类目）→ 二级（语义脉络）→ 三级（关键词）
├── 工作
│   ├── 调试
│   │   ├── WebSocket
│   │   └── 端口占用
│   └── 开发
│       ├── React
│       └── Node.js
├── 学习
│   ├── 网络
│   │   ├── TCP
│   │   └── HTTP
│   └── 算法
│       └── 排序
└── 项目
    ├── 需求
    └── 设计
```

### 标签使用建议

**一级标签（大类目）**:
- 工作: 工作中遇到的问题和解决方案
- 学习: 学习笔记和知识点
- 项目: 项目相关的内容
- 个人: 个人记录和想法

**二级标签（语义脉络）**:
- 调试: 问题排查和调试方法
- 开发: 开发技巧和最佳实践
- 优化: 性能优化相关
- 部署: 部署和运维相关

**三级标签（关键词）**:
- 具体的技术、工具、概念
- 例如: WebSocket、React、Docker

## 🔧 高级功能

### 1. 自然语言查询

支持大白话查询，系统会自动理解你的意图：

```bash
# 保存知识
/kb chat 保存一下：WebSocket超时的解决方法，标签是工作、调试、WebSocket

# 搜索知识
/kb ask 帮我找找关于React性能优化的知识
/kb ask 我记得有个调试问题的解决方法

# 列出知识
/kb chat 列出所有工作相关的知识

# 查看统计
/kb ask 看看统计信息
```

### 2. 智能去重

系统会自动去除重复内容：
- 相同内容只保留一份
- 保留使用次数更多的版本
- 自动清理低质量重复

### 3. 相关性评分

搜索结果会显示相关性分数：
- `[80%匹配]` - 高度相关
- `[60%匹配]` - 中等相关
- `[40%匹配]` - 可能相关

## ❓ 常见问题

### Q: 为什么搜索结果少了？
A: v3.1 版本引入了智能去重，相同内容会自动合并。实际内容数量不变，只是去除了重复项。

### Q: 如何查看完整内容？
A: 从目录中找到你感兴趣的条目，记录下 `🆔` 后面的 ID，然后使用 `/kb get <ID>` 查看详情。

### Q: 可以修改已保存的知识吗？
A: 暂不支持直接修改。建议删除旧的知识，然后重新保存。

### Q: 标签可以修改吗？
A: 暂不支持。建议在保存时仔细选择标签。

### Q: 如何导出知识库？
A: 暂不支持导出功能。数据存储在 `~/.claude/data/knowledge/knowledge.db`。
