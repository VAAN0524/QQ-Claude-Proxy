## 🔍 根本原因分析

### 问题1：重复消息

**原因**：
1. **CLI Session Manager** - stream-json 解析可能有重复事件
2. **ProgressTracker** - 虽然有去重逻辑，但可能没覆盖所有场景
3. **网关发送机制** - 没有全局去重

**流程**：
```
AI响应 → ProgressTracker → QQ Gateway → 发送
                          ↑ 重复
AI响应 → ProgressTracker → QQ Gateway → 发送
```

### 问题2：图片附件处理失败

**原因**：
1. **QQ图片URL需要鉴权** - 无法直接下载
2. **下载失败但继续执行** - 没有检测和处理
3. **Agent自己编造路径** - 没有检测到图片，使用旧任务路径

**流程**：
```
QQ消息(图片) → Agent → 下载图片 → 失败 ❌
                          ↓
                      继续执行
                          ↓
                      没有检测到图片
                          ↓
                      编造旧路径 ❌
```

## ✅ 修复方案

### 修复1：全局消息去重器

**文件**: `src/utils/message-deduplicator.ts`

**功能**：
1. 记录最近发送的消息（5秒窗口）
2. 检查重复内容
3. 自动清理过期记录
4. 智能内容哈希（忽略动态内容）

**集成**：`src/channels/qqbot/index.ts`
```typescript
// 发送前去重检查
const deduplicator = getGlobalDeduplicator();
if (deduplicator.isDuplicate(targetUserId, content, groupId)) {
  logger.warn(`阻止重复消息`);
  return;
}
```

**效果**：防止同一内容在5秒内重复发送

### 修复2：图片附件处理增强

**文件**: `src/agent/index.ts`

**功能**：
1. 检测图片下载失败
2. 明确告知用户图片无法获取
3. 提供替代方案
4. 不再编造图片路径

**代码**：
```typescript
try {
  const storedFile = await this.storage.storeFromUrl(att.url, att.filename);
  storedFiles.push(storedFile);
} catch (error) {
  // 图片下载失败时，明确告知用户
  if (att.type.startsWith('image/')) {
    return {
      content: `❌ 无法获取您发送的图片（QQ图片URL可能需要鉴权）。

💡 请尝试以下方法：
1. 将图片保存到本地，然后发送本地文件路径
2. 或者将图片放到项目目录，告诉我路径
3. 或者描述图片内容，我尽力帮您处理`
    };
  }
}
```
