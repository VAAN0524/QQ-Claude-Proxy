# 场景化叙事配图生成指南

**核心原则：有人物、有场景、有故事、有动态感**

---

## ❌ 错误的配图风格（禁止）

**信息图风格：**
- 左右对比图（A vs B）
- 静态数据展示
- 纯物体/图标展示
- 抽象概念图

**实际效果：**
- 空洞单调
- 未准确传达信息
- 像PPT配图，不像公众号文章
- 毫无创意

---

## ✅ 正确的配图风格：场景化叙事

### 核心要素

1. **人物**：卡通风格的人物（程序员、创业者、家长、上班族等）
2. **场景**：具体的环境（办公室、家庭、咖啡厅、城市等）
3. **动作**：具体的行为（写代码、点外卖、叠衣服、看屏幕等）
4. **AI形式**：屏幕、机器人、全息投影、悬浮界面
5. **动态元素**：悬浮屏幕、连接线、代码流、光线效果
6. **叙事信息**：通过场景元素自然传达信息，不是硬贴数据

---

## 完整示例：DeepSeek-V4编程能力

### 原文章节内容

```markdown
## 01 DeepSeek-V4即将上线

DeepSeek这个团队有点意思。2023年才成立，现在已经成了OpenAI最头疼的对手之一。
3月3日发布的V2版本用236B参数的MoE架构，性能直追GPT-4o，推理成本却只有对方的1/30。
GitHub星标几天就破了万，中文社区讨论量超过15万。

现在V4要来了。梁文锋这次主攻长期记忆和编码能力升级。
简单说，就是让AI能记住更多上下文，写代码更顺手。
```

### 场景化叙事配图设计

**中心思想**：程序员使用DeepSeek-V4写代码，效率提升10倍，代码质量95分

**叙事场景**：
```
时间：深夜，程序员在办公室写代码
主角：年轻程序员（戴眼镜、穿连帽卫衣、表情专注但轻松）
环境：现代化办公室，落地窗外是城市夜景（暗示工作到很晚）
AI形式：电脑屏幕 + 悬浮的AI助手界面

动态元素：
1. 代码流：蓝色的代码行从AI大脑流向代码编辑器
2. 悬浮屏幕1："代码质量：95分"（半透明）
3. 悬浮屏幕2："效率提升10倍"（半透明）
4. 悬浮屏幕3："上下文：1M tokens"（半透明）
5. AI大脑：发光效果，显示"理解中..."

生活细节：
- 桌上一杯咖啡（已经凉了）
- 几本技术书和便利贴
- 电脑右下角时钟：显示"23:30"
- 程序员头顶：思维气泡"这么快就写完了？"

叙事信息（自然融入）：
- AI大脑旁的对话气泡："我理解你的意图了"
- 进度条：100%，绿色对勾
- 墙上GitHub星标图标："⭐ 10,000+"

色彩：
- DeepSeek蓝：#1E88E5
- 办公室暖光：#FFD54F（台灯）
- 背景：深色夜景 #263238

构图：
- 程序员在中心稍偏左
- 电脑屏幕在中间
- AI大脑在屏幕上方
- 悬浮屏幕环绕排列
```

### 对比：信息图风格 vs 场景化叙事

| 维度 | 信息图风格（❌） | 场景化叙事（✅） |
|------|----------------|----------------|
| 主体 | 左侧DeepSeek，右侧GPT-4o | 程序员使用DeepSeek写代码 |
| 构图 | 左右对称对比 | 中心叙事场景 |
| 人物 | 无 | 卡通程序员 |
| 场景 | 无 | 办公室、夜景 |
| 动作 | 无 | 写代码、咖啡凉了 |
| 信息展示 | 底部数据卡片 | 悬浮屏幕、对话气泡 |
| 情感 | 冷冰冰 | 温馨、有生活气息 |
| 故事性 | 无 | 有完整的故事场景 |

## 完整的 Image Skill Prompt 模板

```
使用 Image skill 生成章节配图：

场景类型：场景化叙事插画（必须有人物和场景）
尺寸：1920×1080

主角设计：
- 人物类型：[程序员/创业者/家长/上班族]
- 外貌特征：[年龄、服装、表情、配饰]
- 表情：[专注、轻松、惊喜、困惑等]
- 动作：[具体在做什么]

环境场景：
- 地点：[办公室/家庭/咖啡厅/城市街道]
- 时间：[白天/夜晚/黄昏]
- 氛围：[温馨/忙碌/轻松/紧张]
- 背景：[窗外景色、室内装饰]

AI表现形式：
- 形式：[电脑屏幕/悬浮界面/机器人/全息投影]
- 位置：[在场景中的位置]
- 状态：[正在工作、思考、回应]

动态元素（必须有2-3个）：
- 悬浮屏幕1：[具体内容，半透明]
- 悬浮屏幕2：[具体内容，半透明]
- 流动元素：[代码流/光线/粒子效果]
- 连接线：[连接各个悬浮元素]

叙事信息（通过场景元素传达）：
- 对话气泡：[人物的想法]
- 屏幕文字：[具体的信息/数据]
- 环境细节：[时钟、日历、文件、标签]
- ❌ 不要：底部数据卡片、硬贴的数字

色彩方案：
- 主色：[具体的颜色或氛围]
- 强调色：[用于什么元素]
- 背景：[环境色]

风格：
- 扁平插画风格（参考 Dribbble/Behance）
- 现代、温馨、有生活气息
- 不要冷冰冰的科技感

构图：
- [具体的布局说明]
- 人物在什么位置
- AI元素如何安排
```

## 执行检查清单

生成配图前必须确认：
- [ ] 是否有卡通人物？
- [ ] 是否有具体场景？
- [ ] 是否有具体动作？
- [ ] 是否有动态元素（悬浮屏、连接线）？
- [ ] 信息是否自然融入场景？
- [ ] 是否有生活化细节？
- [ ] 是否有故事性？

如果任何一项是"否"，必须重新设计。
