# WeChat Publisher Skill 重大更新

**更新日期**: 2026-03-16
**更新原因**: 用户反馈配图质量差、图片嵌入失败

---

## 更新概述

针对用户反馈的两个核心问题，对 `wechat-publisher` skill 进行了全面重构：

1. **图片嵌入问题**：草稿箱中仍出现 `![配图](images/xxx.png)` 文本
2. **配图质量差**：生成抽象图形、盒子箱子，无法传达中心思想

---

## 问题1：图片嵌入失败

### 根本原因
- Skill 文档中未强制要求使用修复后的脚本
- 缺少发布前的验证清单
- 流程描述不够具体

### 解决方案

#### 1. 强制使用修复后的脚本

**更新位置**: Step 4（上传草稿箱）

```bash
# ✅ 正确：使用修复后的脚本
python scripts/upload-to-wechat-fixed.py

# ❌ 错误：不要使用旧脚本
python scripts/upload-to-wechat-draft.py      # 有bug
python scripts/upload-to-wechat-correct.py    # 有bug
```

#### 2. 添加发布前验证清单

**新增内容**：
```
Step 4.4: 验证清单（发布前必须确认）
├── [ ] article.md 文件完整
├── [ ] cover.png 封面图已生成
├── [ ] 所有章节配图已生成
├── [ ] 图片验证报告已完成
├── [ ] 所有图片评分 ≥ 80分
├── [ ] 已执行 upload-to-wechat-fixed.py 脚本
├── [ ] 获得了有效的 media_id
├── [ ] 在公众号后台预览草稿
├── [ ] 所有图片在草稿中正确显示
├── [ ] 图片清晰无模糊
└── [ ] 图片位置居中对齐
```

#### 3. 添加问题排查指南

**新增内容**：
- 问题1：草稿中显示 Markdown 文本 → 使用正确的脚本
- 问题2：图片显示红叉 → 检查网络，重新上传
- 问题3：只有封面图 → 检查文件名匹配
- 问题4：图片模糊 → 重新生成高清图片

## 问题2：配图质量差

### 根本原因
- 缺少具体的视觉转译指导
- Prompt 构建过于抽象
- 缺少质量评分标准
- 缺少强制性的视觉验证流程

### 解决方案

#### 1. 配图质量铁律

**新增内容**：
```
❌ 绝对禁止：抽象几何图形、盒子箱子、无意义装饰
❌ 绝对禁止："科技感背景"、"抽象网络"、"神秘图案"
✅ 必须包含：具体可识别的视觉元素
✅ 必须传达：章节的核心观点或数据
```

#### 2. 详细的视觉转译流程

**新增内容**：完整的 Step 3.2 视觉转译流程

```
Step 3.2: 视觉转译（关键步骤）
├── 将中心思想拆解为2-3个视觉元素
├── 为每个元素设计具体表现形式
│
│   示例：中心思想 = "成本暴跌128倍"
│   ├─ 视觉元素1：价格标签
│   │   └─ 左侧$9999标签，右侧$78标签，用箭头连接
│   ├─ 视觉元素2：下降曲线
│   │   └─ 陡峭的下降曲线，标注"-128倍"
│   └─ 视觉元素3：应用场景
│       └─ 企业使用AI的具体场景（工厂/办公室）
│
├── 确定色彩方案
└── 确定构图
```

#### 3. 视觉转译对照表

**新增内容**：详细的正确/错误对照

| 中心思想类型 | ✅ 正确视觉转译 | ❌ 错误视觉转译 |
|-------------|----------------|---------------|
| 成本下降 | 价格标签破碎、下降曲线、天平倾斜 | 抽象箭头、几何图形、科技背景 |
| 技术突破 | 具体技术场景（芯片/代码/机器人） | 神秘光效、抽象网络、无意义图案 |
| 市场竞争 | 对立双方logo、跑道的赛跑、天平对比 | 模糊剪影、抽象对撞、几何方块 |

#### 4. 详细的评分标准

**新增内容**：100分制评分系统

```
评分标准（总分100分）：

├── 内容匹配度（40分）
│   ├── 40分：完全匹配，精准传达中心思想
│   ├── 30分：基本匹配，传达了主要观点
│   ├── 20分：部分匹配，需要文字辅助理解
│   └─ 10分以下：不匹配，需要重新生成
│
├── 视觉具体性（30分）
│   ├── 30分：有具体可识别的元素（不是抽象图形）
│   ├── 20分：有具体元素但不够清晰
│   ├── 10分：有具体元素但模糊难辨
│   └─ 10分以下：抽象图形，无法识别
│
├── 信息传达（20分）
│   ├── 20分：清晰传达关键信息/数据
│   ├── 15分：传达了部分信息
│   ├── 10分：信息模糊需要猜测
│   └─ 10分以下：没有传达有效信息
│
└── 技术质量（10分）
    ├── 10分：清晰、无变形、构图合理
    ├── 7分：基本清晰，有轻微问题
    ├── 5分：有模糊或变形
    └─ 5分以下：质量差，无法使用

判定标准：
├── 90分以上：优秀，直接使用
├── 80-89分：良好，可以使用
├── 70-79分：及格，记录改进点
└── 70分以下：不合格，必须重新生成
```

#### 5. Image Skill Prompt 构建模板

**新增内容**：详细的 Prompt 构建指南

```
✅ 正确示例：
类型：信息图对比图
主体：左侧巨大的价格标签$9999，右侧小标签$78，中间下降箭头标注128倍
背景：简洁的浅灰色，不要抽象图案
色彩：深蓝色主调，绿色强调价格，白色文字
风格：扁平插画风格，清晰易读
文字：大号'成本暴跌128倍'作为标题
构图：左右对比，清晰展示价格差异

❌ 错误示例：
生成科技感配图，抽象风格
展示AI变革的未来感
```

## 使用建议

### 执行公众号文章任务时

1. **撰写章节后**：
   - 填写视觉转译模板
   - 提炼中心思想
   - 设计视觉元素

2. **生成配图前**：
   - 构建详细的 Image Skill Prompt
   - 参考正确/错误对照表
   - 避免使用抽象词汇

3. **生成配图后**：
   - 使用 Read 工具验证图片
   - 按100分制评分
   - 70分以下必须重新生成

4. **发布前**：
   - 使用 `upload-to-wechat-fixed.py` 脚本
   - 完成验证清单
   - 在公众号后台预览确认
